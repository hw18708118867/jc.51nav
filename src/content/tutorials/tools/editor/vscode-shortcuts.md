同样是改一百行相似的代码，有人鼠标点了一百次，有人按了三个键就搞定。差距就在**高效编辑技巧**。这一篇挑出最值回票价的几招，练熟后你会发现"手根本不想离开键盘"。

把编辑器想成钢琴：只会一个指头敲，曲子弹得又慢又累；熟了和弦与琶音，同样的乐谱行云流水。下面这些就是你的"和弦"。

## 多光标：一处改，处处改

最常用的一招。你想把文档里多处相同的词一起改：

- **选中一个词，按 `Ctrl/Cmd + D`**：每按一次，选中下一个相同的词，同时出现多个光标，打字时全部一起改。
- **`Ctrl/Cmd + Shift + L`**：一次性选中所有相同的词，全部加光标。
- **`Alt + 点击`**（macOS 是 `Option + 点击`）：在任意多处手动放置光标。

场景：变量名 `user` 要改成 `account`，`Ctrl/Cmd + D` 连按几次，输入新名，全文同步改完。

## 列选择：竖着框选

有时要改的是"每一行的同一列"，比如给十行前面都加个 `- `：

- 按住 `Shift + Alt`（macOS `Shift + Option`）再拖动鼠标，就能竖着框选一个矩形区域，输入内容会同时写进每一行。

## 快速跳转文件

- **`Ctrl/Cmd + P`**：弹出"转到文件"框，输入文件名（甚至模糊匹配，如 `vcset` 能命中 `vscode-extensions`）回车即开。比在文件树里翻快十倍。
- **`Ctrl/Cmd + Shift + P`**：命令面板，前面讲过，搜一切命令。

## 符号与行号跳转

- **`Ctrl/Cmd + Shift + O`**：列出当前文件的所有函数/类（符号），输入名字直接跳过去，长文件导航神器。
- **`Ctrl/Cmd + G`**：输入行号跳转，比如 `45` 直接去第 45 行。
- **`Ctrl/Cmd + 单击`**：按住 Ctrl 点函数名，跳到它的定义处（要看实现就靠它）。
- **`Ctrl/Cmd + -`**：跳回刚才的位置（看完定义返回）。

## 移动与选择

- **`Alt + 上下方向键`**：把当前行整体上下移动，不用剪切粘贴来重排。
- **`Shift + Alt + 上下`**：复制当前行到上/下。
- **`Ctrl/Cmd + 左右`**：按"词"左右跳光标（比一个字符一个字符快）。配合 `Shift` 就是按词选择。
- **`Ctrl/Cmd + 退格`**：删掉光标前的一个词（不是一个个字符删）。

## 编辑器布局

- **`Ctrl/Cmd + \`**：左右拆分编辑器，对照两个文件。
- **`Ctrl/Cmd + 1/2/3`**：在拆分的多个编辑器间切换焦点。
- **`Ctrl/Cmd + B`**：隐藏/显示左侧侧边栏，腾出屏幕看代码。
- **`Ctrl/Cmd + J`**：切换底部面板（终端）显隐。

## 常见新手坑

- **只会 `Ctrl+C / Ctrl+V` 逐行改**：遇到批量改名、批量加前缀，应优先想"多光标"或列选择。
- **在文件树里一层层翻**：文件多了找不到，`Ctrl/Cmd + P` 模糊搜索快得多。
- **忘了"跳回"快捷键**：`Ctrl/Cmd + -` 能回到跳转前的位置，省得手动滚回去。
- **不知道 `Alt + 上下` 移行**：重排代码还在用剪切粘贴，慢且易错。

## 小测验

- 问题1：想把文档里所有 `user` 一次性改成 `account`，用什么？答案：`Ctrl/Cmd + Shift + L` 全选相同词加多光标，再输入。
- 问题2：模糊打开文件 `vscode-extensions.md` 用什么快捷键？答案：`Ctrl/Cmd + P` 输入文件名。
- 问题3：竖着给多行同时加内容用什么操作？答案：按住 `Shift + Alt`（macOS `Shift + Option`）拖动做列选择。

## 更多实战：用快捷键重命名一批变量

假设你把 `usr` 改成 `user`，全程键盘：

1. 双击选中一个 `usr`，按 `Ctrl/Cmd + Shift + L` 全选所有 `usr`，同时出现多光标。
2. 直接输入 `user`，全文同步改完。
3. 按 `Ctrl/Cmd + S` 保存。

对比鼠标逐个改，这波操作省下几十次点击。练熟多光标，你会发现自己越来越不想碰鼠标。

## 把常用快捷键钉进肌肉记忆

新手别贪多，先死记这三组，够用很久：

- `Ctrl/Cmd + P`：找文件（最高频）。
- `Ctrl/Cmd + Shift + P`：找命令。
- `Alt + 上下`：移动整行。

等这三组变成本能，再慢慢加 `Ctrl/Cmd + D`、`Ctrl/Cmd + \` 等。

## 自测：你真的懂了吗

- 想全文把 `user` 一次性改成 `account` 用什么？答案：`Ctrl/Cmd + Shift + L` 全选加多光标再输入。
- 模糊打开文件用什么快捷键？答案：`Ctrl/Cmd + P` 输入文件名。
- 竖着给多行同时加内容用什么？答案：`Shift + Alt` 拖动做列选择。

## 常见认知误区

快捷键这块，新手最容易"学了一堆用不上"。误区一是"想一次性记住所有快捷键"——结果哪个都没记住，应该先死磕最高频的三组：`Ctrl/Cmd + P` 找文件、`Ctrl/Cmd + Shift + P` 找命令、`Alt + 上下` 移行，练成本能再加别的。误区二是"还在文件树里一层层翻文件"——文件多了根本找不到，`Ctrl/Cmd + P` 模糊搜索快十倍。误区三是"改长命令用退格删到头"——应该用 `Ctrl + A` 跳行首、`Ctrl + W` 删词，效率天差地别。误区四是"复制用 Ctrl + C"——在终端里这俩字符是中止程序，复制得用 `Ctrl + Shift + C`，这是最多人踩的坑。误区五是"忘了跳回快捷键"——`Ctrl/Cmd + -` 能跳回跳转前的位置，看完函数定义不用手动滚回去。快捷键不是炫技，是每天省下的真实时间。

## 这一篇你该记住的

- 多光标：`Ctrl/Cmd + D` 逐个加、`+ Shift + L` 全加、`Alt + 点击` 手动加。
- 列选择：`Shift + Alt` 拖动，竖着批量编辑。
- 跳转：`Ctrl/Cmd + P` 搜文件、`+ Shift + O` 搜符号、`+ G` 跳行号、`Ctrl + 单击` 跳定义。
- 移动：`Alt + 上下` 移行、`Shift + Alt + 上下` 复制行。
- 布局：`Ctrl/Cmd + \` 拆分、`+ B` 收侧栏、`+ J` 收面板。
