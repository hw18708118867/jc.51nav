前面几篇把 VS Code 的编辑、插件、快捷键、调试都过了一遍。这一篇收个尾：把"终端"和"重构"这两块补上，你就能在编辑器里完成"写代码 → 跑命令 → 改结构"的完整闭环，几乎不用切出 VS Code。

把 VS Code 想成你的"作战指挥室"：编辑区是指挥台，集成终端是通讯频道，重构工具是工程兵——都在一个房间里，调动起来毫不费劲。

## 集成终端：不用再开独立窗口

底部面板里的**终端（Terminal）** 就是集成终端，它就是个嵌在编辑器里的命令行：

- 打开：`Ctrl/Cmd + \``（反引号键）或菜单"终端 → 新建终端"。
- 用法和你平时用的系统终端一模一样：能跑 `npm install`、`git commit`、`python app.py` 等任何命令。
- 好处：命令和代码在同一个窗口，报错时直接点终端里的文件路径跳回对应代码行。

```bash
# 在集成终端里跑项目
npm install
npm run dev
```

终端支持多开标签（`+` 新建）、分屏（右键"拆分终端"），还能选不同 shell（bash、zsh、PowerShell）。

## 重构：安全地改代码结构

**重构**是"不改功能、只改结构"的代码整理。手动改容易漏，VS Code 提供安全的自动化重构：

### 重命名（最常用）

你有个变量 `usr` 想改成 `user`，全项目几十处引用。手动改会漏，用重命名：

- 把光标放在变量上，按 `F2`（或右键"重命名符号"），输入新名回车。
- VS Code 会**把所有引用处一起改掉**，且只改这个变量，不会误伤同名字符串。

这是日常最高频的重构，比全局查找替换安全得多（查找替换可能改到不该改的地方）。

### 提取函数 / 变量

一段逻辑重复出现，想抽成函数复用：

1. 选中那段代码。
2. 快捷键 `Ctrl/Cmd + Shift + R` 或右键"重构"→ "提取到函数"。
3. VS Code 自动生成函数，并把原处的代码换成函数调用。

同理可"提取到常量"，把魔法数字变成有意义的命名常量。

### 移动 / 内联

- **移动到新文件**：把某个类/函数独立成文件，引用自动更新。
- **内联**：把一次函数调用展开回原处（反向重构）。

## 一站式开发闭环

有了前面所有能力，你的典型一天可以是这样：

```text
Ctrl/Cmd + P 打开文件 → 多光标批量改 → F5 调试跑起来 →
底部终端 npm run dev 看效果 → F2 重命名理结构 → Ctrl/Cmd + Shift + P 搜命令收尾
```

全程键盘为主、鼠标为辅，上下文不丢失，状态不切换。

## 常见新手坑

- **终端里 `cd` 来 `cd` 去找不到项目**：其实用 `code .` 在正确目录打开项目，终端默认就在项目根。
- **用全局替换改变量名**：容易误伤同名文本，应用 `F2` 重命名符号。
- **重构前不提交**：重构改动了多处，一旦改错难回滚。重构前先 `git commit` 存个档。
- **终端堆积太多标签**：久了分不清哪个在跑什么，定期关掉不用的。

## 小测验

- 问题1：重命名一个变量，为什么用 F2 比全局查找替换好？答案：F2 只改该符号的所有引用，不会误伤同名字符串，更安全。
- 问题2：想把一段重复代码抽成函数，用什么重构？答案：选中代码后"提取到函数"。
- 问题3：集成终端相对独立终端窗口的最大好处？答案：命令和代码同窗，报错可点路径跳回代码。

## 更多实战：一个真实开发回合

假设你要修一个接口 bug：

1. `Ctrl/Cmd + P` 搜 `user.api.js` 打开。
2. 多光标改两处字段名。
3. `F5` 调试跑起来，断点看请求参数。
4. 底部终端 `npm run dev` 起服务看效果。
5. 改完 `F2` 把变量 `resp` 重命名为 `response`，全项目引用同步。
6. `Ctrl/Cmd + Shift + P` 搜命令收尾。

全程不离开 VS Code，上下文不丢、状态不切。

## 重构前先存档

重构（重命名、提取函数）会改动多处文件，一旦改错难手动还原。所以重构前先 `git commit` 存个档，万一改砸了 `git reset` 或 revert 就能回退。这是"敢重构"的底气。

## 常见认知误区

集成终端与重构这块，新手常犯几个错。误区一是"终端里 `cd` 来 `cd` 去找不到项目"——其实用 `code .` 在正确目录打开项目，集成终端默认就在项目根，不用乱跑。误区二是"用全局替换改变量名"——容易误伤同名字符串，应该用 `F2` 重命名符号，它只改该符号的所有引用，安全得多。误区三是"重构前不提交"——重构会改动多处文件，一旦改错难手动还原，先 `git commit` 存个档，改砸了能回退，这是敢重构的底气。误区四是"终端堆积太多标签"——久了分不清哪个在跑什么，定期关掉不用的。误区五是"觉得集成终端不如独立终端"——其实功能一样，还多了"报错点路径跳回代码"的便利，日常开发用集成终端上下文不丢失，体验更好。把编辑、终端、调试、重构都收进 VS Code，你就拥有了不打断心流的一站式工作台。

## 这一篇你该记住的

- 集成终端 `Ctrl/Cmd + \`` 打开，用法同系统终端，报错可点路径回跳。
- 重命名 `F2`：安全改所有引用，优于全局替换。
- 提取函数/变量：`Ctrl/Cmd + Shift + R` 把重复逻辑收成复用单元。
- 重构前先 `git commit` 存档，改错了能回滚。
- 编辑、终端、调试、重构都在 VS Code 内，形成无切换开发闭环。
