VS Code 本身像一把基础瑞士军刀，真正让它变强的，是你往上面装的"刀头"——**插件（Extension）**。但插件装多了也会拖慢启动、互相打架。这一篇帮你挑出最值得装的几类，避开"装了一堆用不上"的坑。

把插件想成手机的 App：系统自带打电话发短信（基础编辑），但导航、支付、笔记得靠 App（插件）补上。装对了事半功倍，装多了手机卡顿（编辑器变慢）。

## 怎么装插件

- 点活动栏的"扩展"图标（方块拼图），或 `Ctrl/Cmd + Shift + X` 直接打开扩展视图。
- 搜索名字 → 点"安装" → 多数即时生效，少数要重载窗口。
- 卸载同样在这里，点已安装插件旁的齿轮。

## 第一类：语言与框架支持

VS Code 对 JS/TS 有基础支持，但其他语言常需补插件才智能：

- **Python**：装 `Python`（微软官方），获得运行、调试、智能提示、虚拟环境识别。
- **Go**：装 `Go`（官方），含格式化、补全、测试。
- **Java**：装 `Extension Pack for Java`（一包搞定）。
- **前端框架**：`Vue`、`ES7+ React Snippets` 等提供代码片段和语法高亮。

原则：**写什么语言，就装对应的官方/社区语言插件**，否则没有补全和报错提示，像在记事本里写代码。

## 第二类：代码规范与质量

- **Prettier**：最流行的代码格式化工具，保存时自动把代码排得整整齐齐，团队统一风格就靠它。
- **ESLint**：JavaScript/TypeScript 的"代码警察"，实时标出潜在错误和坏味道（如未使用的变量）。
- **EditorConfig**：用 `.editorconfig` 文件统一不同编辑器间的缩进、换行等基础风格。

这类插件让"代码长得好看、少出错"自动化，新手尤其该装。

## 第三类：Git 增强

- **GitLens**：在每一行代码旁显示"谁、什么时候、哪次提交改的"，追查"这行为啥这样写"的神器。
- **Git Graph**：用图形化界面看分支树和提交历史，比命令行 `git log --graph` 直观。

## 第四类：效率与体验

- **Error Lens**：把报错信息直接显示在代码行尾，不用悬停才看得到。
- **Todo Tree**：自动收集代码里的 `TODO`、`FIXME` 注释，汇成一个清单，方便回头清理。
- **Path Intellisense**：写文件路径时自动补全，少拼错。
- **Live Server**：写前端时一键起本地服务器并自动刷新，改完立刻看效果。
- **Docker / Remote-SSH**：容器开发、远程连服务器写代码时必备。

## 怎么挑插件不踩坑

- **看安装量和评分**：百万级安装、4–5 星的基本靠谱。
- **看最近更新**：一年没更新的慎装，可能不兼容新版本 VS Code。
- **看发布者**：带"已验证"徽章的官方/知名团队更可信。
- **别贪多**：只装当下用得上的，卡了就卸掉不常用的。
- **同步设置**：登录 Microsoft 或 GitHub 账号，插件和配置能跨设备同步，换电脑不重配。

## 常见新手坑

- **插件装一堆从不删**：启动变慢、快捷键冲突。定期清理。
- **Prettier 和 ESLint 打架**：两者规则冲突时格式化乱跳。通常让 Prettier 管格式、ESLint 管逻辑，并在 ESLint 里关掉与格式重叠的规则。
- **装了语言插件却没生效**：可能没装对应运行环境（如装了 Python 插件但没装 Python 解释器）。
- **盲目装"网红"插件**：别人用着香不代表你用得上，按需取用。

## 小测验

- 问题1：写 Python 该装什么插件获得补全和调试？答案：微软官方的 `Python` 扩展。
- 问题2：Prettier 和 ESLint 分工有何不同？答案：Prettier 管代码格式（排版），ESLint 管代码质量和潜在错误。
- 问题3：挑插件时最该看哪两个指标？答案：安装量和最近更新时间（太旧的可能不兼容）。

## 更多实战：一套推荐起步组合

如果你不知道装什么，这套组合几乎适合所有人：

- 写前端：`Prettier` + `ESLint` + `Live Server` + `Path Intellisense`。
- 写 Python：`Python`（官方）+ `Pylance` + `GitLens`。
- 通用提效：`Error Lens` + `Todo Tree` + `Git Graph`。

装完别急着加更多，用上一周，哪个真天天用就留，其余卸掉。

## 插件冲突怎么排查

如果发现保存后格式乱跳、或某个功能时灵时不灵，多半是插件打架：

1. 命令面板搜 `Extensions: Show Enabled Extensions` 看已启用列表。
2. 怀疑某个插件，先禁用它，重启窗口看问题是否消失。
3. 确认后要么卸掉，要么在设置里关掉它冲突的那项功能。

## 常见认知误区

插件这块最容易"装了一堆用不上"。误区一是"网红插件都装"——别人用着香不代表你用得上，按需取用，装了不用的只会拖慢启动。误区二是"Prettier 和 ESLint 重复装"——它们分工不同，Prettier 管格式排版、ESLint 管代码质量和潜在错误，不是二选一，而是配合：让 Prettier 管格式、ESLint 管逻辑，并在 ESLint 里关掉与格式重叠的规则，避免保存时格式乱跳。误区三是"插件装完不验证"——装了语言插件却没装对应运行环境（如装了 Python 插件但没装 Python 解释器），补全和调试都不生效，还以为是插件问题。误区四是"忽略更新时间"——一年没更新的插件可能不兼容新版本 VS Code，挑插件时看安装量和最近更新，太旧的要谨慎。误区五是"不敢卸"——用不上的定期清理，保持轻快。

## 这一篇你该记住的

- 插件是 VS Code 的"刀头"，按需装、别贪多。
- 语言插件（Python/Go/Java）给补全和调试；Prettier/ESLint 管格式与质量。
- GitLens、Git Graph 让版本历史可视化；Error Lens、Todo Tree 提效。
- 挑插件看安装量、评分、更新时间、发布者可信度。
- 登录账号同步插件与设置，换设备零成本。
