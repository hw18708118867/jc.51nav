开发者的成长，一半靠写代码，一半靠"记"。你今天查到的某个命令、踩过的某个坑、调通的一段配置，过两周大概率忘。如果每次都重新搜，时间全耗在"找已知答案"上。笔记工具和代码片段管理器，就是帮你把这些零散知识攒起来，变成随取随用的"第二大脑"。这一篇讲怎么搭。

把笔记和片段想成你的"外挂硬盘"：脑子只记"我存过这个"和"大概在哪"，细节交给工具。需要时点一下就调出来，不用重新想、重新搜。

## 为什么需要它们

- **人脑善忘**：踩过的坑不记，下次还踩；查过的命令不存，下次还搜。
- **重复造轮子**：一段好用的正则、一个 Docker 启动命令，每次重写既慢又易错。
- **知识碎片化**：答案散在聊天记录、浏览器书签、便签里，要用时找不到。

## 笔记工具：沉淀知识

笔记适合存"长一点的、需要理解的内容"——某次排错的过程、某个概念的理解、读书笔记。

常见选择：

- **Notion**：全能型，文档、表格、数据库一体，团队协作强，免费个人够用。
- **Obsidian**：本地优先、Markdown 文件，双向链接把笔记连成网络，隐私好、可长期持有。
- **Typora / 熊掌记**：轻量纯写作， distraction-free。
- **语雀 / 飞书文档**：国内团队常用，中文体验好。

选哪种看你习惯：重视隐私和可迁移选 Obsidian（文件在你硬盘）；重视协作和模板选 Notion。

### 笔记怎么记才有用

- **用自己的话写**：抄别人的没用，复述一遍才真懂。
- **记"触发场景"**：别只写"怎么做"，写"什么时候用、为什么这么用"。比如不写"用 grep -r"，写"要在整个项目找某个函数定义时用 grep -rn"。
- **加标签和链接**：把相关笔记连起来，形成知识网而非孤岛。
- **定期回顾**：攒了一堆不看等于没攒，每周翻翻旧的，温故知新。

## 代码片段工具：复用代码

片段适合存"短而常用的代码/命令"——一段正则、一个 SQL 模板、一个 Docker 命令、一个常用 Git alias。

常见做法：

- **VS Code 用户片段**：在设置里定义自己的代码片段，输入触发词自动展开。比如输入 `logd` 展开成 `console.log('debug:', ...)`。
- **专用工具**：`SnippetsLab`（macOS）、`CodeExpander`、`massCode` 跨平台，支持分类、搜索、云同步。
- **Raycast / Alfred 片段**：前文提过，把常用片段存进启动器，全局调用。

### 片段怎么组织

- **按语言/场景分类**：正则一类、SQL 一类、Shell 一类，找得快。
- **带占位符**：好的片段留 `{{变量}}` 占位，展开后填具体值，复用不重写。
- **写清用途注释**：半年后你只记得"我存过个 Docker 片段"，得靠注释想起是干啥的。

## 笔记与片段怎么配合

简单说：**片段存"能直接抄的代码/命令"，笔记存"为什么这么写、踩了什么坑"**。两者互补——片段是弹药，笔记是说明书。

## 常见新手坑

- **工具换来换去**：今天 Notion 明天 Obsidian，知识散落多处反而更乱。选定一个深耕。
- **只收藏不回顾**：存了一堆"稍后看"，从不看，等于没存。定期回顾才转化成人脑知识。
- **片段不分类不写注释**：攒多了像垃圾堆，要用时找不到，还不如重写。
- **过度追求工具**：花一下午调笔记软件配置，却没写几条笔记。工具够用就行，内容才是核心。

## 小测验

- 问题1：笔记和代码片段分别适合存什么？答案：笔记存需理解的长内容（排错过程、概念）；片段存短而常用的代码/命令。
- 问题2：为什么记笔记要"用自己的话"？答案：复述才真懂，抄别人的留不进脑子。
- 问题3：重视隐私和可迁移，选哪个笔记工具？答案：Obsidian，本地 Markdown 文件，不依赖厂商。

## 更多实战：攒一个 Docker 启动片段

你常要起一个 MySQL 容器，每次手敲一长串命令容易错。存成片段：

```bash
docker run -d \
  --name mysql-dev \
  -e MYSQL_ROOT_PASSWORD=123456 \
  -p 3306:3306 \
  mysql:8
```

在 VS Code 用户片段里定义触发词 `dkmysql`，输入就展开；或存进 Raycast 片段全局调用。半年后你忘了参数顺序，调出来照样用，不用重新查文档。

## 笔记的例子：记一次排错

别只写"用 grep 找的"，写：

> 场景：服务启动报端口被占。用 `lsof -i :3000` 查到占用进程 PID， `kill -9 PID` 释放。以后遇到端口占用先 lsof 再 kill。

这种带"触发场景 + 做法"的笔记，下次真遇到能直接照做，比干巴巴的命令有用十倍。

## 常见认知误区

笔记和片段这块，最容易"工具玩得溜、内容没攒下"。第一，工具换来换去——今天 Notion 明天 Obsidian，知识散落多处反而更乱，选定一个深耕。第二，只收藏不回顾——存了一堆"稍后看"从不看，等于没存，定期回顾才转化成人脑知识。第三，片段不分类不写注释——攒多了像垃圾堆，要用时找不到，还不如重写，按场景分类、带用途注释才实用。第四，过度追求工具配置——花一下午调笔记软件，却没写几条笔记，工具够用就行，内容才是核心。第五，笔记抄别人的——不复述就留不进脑子，用自己的话写才真懂。第六，混淆笔记和片段的用途——片段存能直接抄的代码命令，笔记存为什么这么写、踩了什么坑，两者互补而非互相替代。把零散知识攒成第二大脑，是开发者复利最高的习惯之一。
