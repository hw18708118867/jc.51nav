你兴冲冲打开教程想敲命令，结果第一步就懵了：虚拟机是啥？装哪个系统？CentOS 和 Ubuntu 有啥区别？这一篇就把"把 Linux 跑起来"这件事讲透，让你不再卡在环境上。

学 Linux 最大的拦路虎不是命令难记，而是"连个能练手的环境都没有"。很多人教程看了一半，因为没环境动手，转头就忘了。所以这一篇的目标很明确：**帮你用最低成本，在自己的电脑上拥有一台能随便折腾的 Linux 机器。**

## 为什么要用虚拟机

直接把 Linux 装成电脑主系统（"双系统"或"单系统"），对新手太不友好——一旦配错网络或驱动，连不上网会很崩溃，而且你日常还得用微信、浏览器，全切到 Linux 很不方便。

更稳妥的做法是用**虚拟机（VM）**：在一台物理电脑里"套"出一台虚拟电脑。你的 Windows/Mac 是"宿主"，虚拟机里跑的 Linux 是"客人"，两者互不干扰。你在虚拟机里把系统玩崩了，点一下"恢复快照"就满血复活，宿主电脑毫发无伤。

打个比方：虚拟机就像在电脑里开了一个"沙盒游戏"，你在里面怎么造、怎么炸都不会影响外面的真实桌面。

## 两种常见虚拟化方案

- **VMware Workstation / VMware Fusion**：老牌商业软件，稳定、功能全。个人学习用免费的 VMware Workstation Player（Windows）或 VMware Fusion（Mac 的个人版）就够了。
- **VirtualBox**：Oracle 出的开源免费虚拟机，跨平台，轻量。新手首选，因为不要钱、够用。

安装过程就是"下一步下一步"，装完打开是一个管理界面，你可以在里面"新建一台电脑"。

## 下载 Linux 镜像（ISO 文件）

Linux 不是"一个系统"，而是一大家子。不同"发行版（Distribution）"就像不同口味的方便面——内核一样，但包装、默认软件、包管理器不同。常见几个：

| 发行版 | 适合人群 | 包管理器 | 特点 |
| --- | --- | --- | --- |
| **Ubuntu** | 绝对新手、桌面用户 | `apt` | 社区大、教程多、开箱即用 |
| **CentOS / Rocky / Alma** | 服务器、学运维 | `yum`/`dnf` | 企业服务器主流，稳定 |
| **Debian** | 追求稳定的服务器 | `apt` | Ubuntu 的"上游"，极稳 |
| **Arch** | 爱折腾的进阶用户 | `pacman` | 滚动更新、高度自定义（新手慎入） |

**新手建议**：想学服务器运维选 CentOS 系（Rocky Linux 是 CentOS 停更后的接班人）；想轻松上手桌面体验选 Ubuntu。本系列后面命令以 CentOS 系（`yum`/`dnf`）和 Ubuntu（`apt`）都会提到，所以两个都了解一下没坏处。

去官网下载一个 `.iso` 文件（这就是"系统安装光盘"的电子版），比如 `ubuntu-22.04-desktop-amd64.iso` 或 `Rocky-9-x86_64.iso`，放到本地。

## 在虚拟机里安装系统（以通用步骤为例）

1. 打开 VirtualBox/VMware，点"新建"，给虚拟机起个名（如 `linux-study`），类型选 Linux，版本选对应发行版。
2. 分配内存（**建议至少 2GB**，即 2048 MB）和硬盘（**建议 20GB 起**）。这些只是"虚拟"资源，不占满你真实硬盘。
3. 在"光驱/启动盘"里挂载刚才下载的 `.iso` 文件。
4. 启动虚拟机，会进入系统安装界面。
5. 安装向导里关键几步：
   - **语言**：选中文（或 English，看你习惯）。
   - **安装类型**：选"清除整个磁盘并安装"（这是虚拟磁盘，清空无所谓）。
   - **设置用户名和密码**：记住你设的密码，后面 `sudo` 要用。
   - **是否安装 SSH 服务**：建议勾上，方便以后用终端远程连。
6. 等进度条走完，重启，拔掉"虚拟光盘"，用你设的账号登录。看到桌面或命令行提示符，恭喜，环境成了！

> 小提示：如果是"最小化安装（Minimal）"，装完只有命令行没有图形界面。学服务器运维反而推荐最小化——逼你习惯敲命令，而且更省资源。新手如果怕黑屏，就选带桌面的版本。

## 第一次登录后该做什么

进系统第一件事，更新一下软件源（相当于手机系统更新）：

```bash
# Ubuntu / Debian 系
sudo apt update && sudo apt upgrade -y

# CentOS / Rocky 系
sudo dnf update -y
```

`sudo` 是"以管理员身份执行"，因为更新系统需要权限。第一次用 `sudo` 会让你输密码（输的时候屏幕不显示星号，正常，输完回车即可）。

然后试试最经典的欢迎命令：

```bash
uname -a      # 查看内核版本
cat /etc/os-release   # 查看发行版信息
```

## 远程连接：用 SSH 更舒服

在虚拟机窗口里敲命令体验一般（不能复制粘贴、屏幕小）。更舒服的是用你宿主机的终端通过 **SSH** 连进去：

```bash
# 在你自己的电脑（宿主）终端里
ssh 用户名@虚拟机IP
```

比如 `ssh student@192.168.1.100`。第一次连会问你"是否信任"，输入 `yes`，再输密码就能进。这样你就能在自己习惯的终端里操作 Linux 了。

> 想知道虚拟机 IP，在 Linux 里敲 `ip addr`（后面网络篇细讲），找 `inet` 后面那串 `192.168.x.x` 就是。

## 常见新手坑

- **虚拟机启动黑屏/卡住**：多半是电脑的"虚拟化"没开。进 BIOS 把 Intel VT-x / AMD-V 开启（不同主板叫法不同，搜一下你电脑型号）。
- **分配内存太大导致宿主卡死**：虚拟机内存别超过你真实内存的一半。
- **忘了 root 密码**：虚拟机里可以用 `sudo passwd root` 重设；进单用户模式也能找回。
- **分不清"宿主机"和"虚拟机"**：你 `ssh` 连的是虚拟机，别在宿主机终端里瞎敲 Linux 命令。

## 这一篇你该记住的

- 用虚拟机练 Linux 最安全：玩崩了能恢复快照，不影响真实电脑。VirtualBox（免费）或 VMware 都行。
- 发行版选 Ubuntu（桌面/新手）或 CentOS 系 Rocky（服务器运维）；不同发行版包管理器不同（`apt` vs `yum`/`dnf`）。
- 安装流程：新建 VM → 分配内存/硬盘 → 挂载 ISO → 按向导装 → 设密码 → 登录。
- 装完先 `apt update`/`dnf update` 更新；用 `ssh 用户@IP` 从宿主机远程连更舒服。
- 新手别怕命令行，最小化安装反而更练手。
